# Project Instructions

Project ini adalah project PHP/Laravel. Fokus pada backend, business logic, database, Eloquent, controller, service, request validation, route, migration, dan seeder.

## Cara Kerja

- Jangan langsung edit file sebelum menjelaskan rencana.
- Baca file terkait dulu sebelum memberi solusi.
- Fokus pada branch aktif. Jangan anggap struktur branch lain sama.
- Jangan hardcode API key, token, password, credential, atau data sensitif.
- Untuk perubahan besar, pecah jadi langkah kecil.
- Jika ada risiko, jelaskan dulu sebelum memberi patch.

## Gaya Jawaban

Gunakan bahasa Indonesia santai, ringkas, tapi tetap jelas.

Saat menjelaskan bug/perubahan, gunakan format ini jika cocok:

Masalah:
Penyebab:
Analogi:
Solusi:
Pro:
Kontra/Risiko:
Langkah aman:

Saya butuh paham alasan teknisnya, bukan cuma disuruh copy-paste. Gunakan analogi sederhana agar konsep lebih mudah nancep.

## Laravel Checklist

Jika relevan, cek:

- route
- controller
- request validation
- model dan relasi
- migration
- seeder
- service class
- observer/event
- blade/view
- query database

Perhatikan potensi bug seperti N+1 query, relasi null, foreign key salah, nama tabel/kolom tidak cocok, validasi kurang, akses role salah, dan pagination/filter/search rusak.

## Command

Gunakan command Windows PowerShell kecuali diminta lain.

Beri peringatan sebelum command yang bisa menghapus data atau mengubah banyak file.

## Tujuan

Bantu memperbaiki kode sekaligus menjelaskan pola pikir backend developer yang benar. Jawaban harus hemat token, tapi tetap edukatif dan pakai analogi.
